# 答题系统 - 完整部署指南

## 概述

本系统部署到公网需要两个服务：
- **Supabase**: 云数据库（PostgreSQL，免费）
- **Render**: 后端托管（Python Flask，免费）

支持5人同时使用，手机和电脑都能访问。

## 第三步：创建 GitHub 仓库

### 3.1 创建新仓库

1. 登录 https://github.com
2. 点击右上角 **+** → **New repository**
3. 填写：
   - **Repository name**: `quiz-system`
   - **Visibility**: Private（选私有，代码不公开）
   - 其他保持默认
4. 点击 **Create repository**

### 3.2 上传代码

你有两种方式上传代码：

**方式A：网页上传（最简单）**

1. 在新创建的仓库页面，点击 **uploading an existing file**
2. 打开你的 `quiz-system` 文件夹，把**除了 `data/` 文件夹里的 .db 文件**之外的所有文件拖进去
3. 注意：`data/supabase_init.sql` 要上传，`data/quiz.db` 不上传
4. 点击 **Commit changes**

**方式B：命令行上传（如果你熟悉Git）**

```bash
cd quiz-system
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/quiz-system.git
git push -u origin main
```

### 3.3 确认

上传完成后，在GitHub仓库页面应该能看到所有文件，包括 `render.yaml`。

## 第四步：部署到 Render

### 4.1 注册 Render

1. 访问 https://render.com
2. 点击 **Get Started**
3. 用 GitHub 账号登录（最方便）
4. 完成注册

### 4.2 创建 Web Service

1. 登录 Render 后，点击右上角 **New +** → **Web Service**
2. 选择 **Configure account** 授权 Render 访问你的 GitHub 仓库
3. 找到 `quiz-system` 仓库，点击 **Connect**

### 4.3 配置服务

填写以下信息（大部分 `render.yaml` 会自动配置好）：

| 配置项 | 值 |
|--------|-----|
| **Name** | `quiz-system`（或你喜欢的名字） |
| **Region** | 选一个离你近的（如 Singapore 或 Oregon） |
| **Runtime** | `Python 3` |
| **Build Command** | `pip install -r requirements.txt && bash build.sh` |
| **Start Command** | `gunicorn app:app --bind 0.0.0.0:$PORT --workers 2` |
| **Instance Type** | Free（免费） |

### 4.4 配置环境变量

在 **Environment Variables** 部分，添加以下变量：

| Key | Value |
|-----|-------|
| `DB_TYPE` | `postgresql` |
| `DB_HOST` | `db.kamjrkunqsltmuikpjri.supabase.co` |
| `DB_PORT` | `5432` |
| `DB_NAME` | `postgres` |
| `DB_USER` | `postgres` |
| `DB_PASSWORD` | `ML&EwmdnR*qCpg3` |

> 注意：`render.yaml` 文件里已经配置了大部分变量，密码需要你手动添加（因为是敏感信息，不会写在代码里）。

### 4.5 开始部署

点击 **Create Web Service**，然后等待部署完成。

- 第一次部署大约需要 3-5 分钟
- 你会看到日志在滚动
- 当显示 **Live** 或 **Deployed** 就说明成功了

### 4.6 获取访问地址

部署成功后，页面顶部会显示你的服务地址，类似：
```
https://quiz-system-xxxx.onrender.com
```

这就是你的公网访问地址，手机和电脑都能打开。

## 常见问题

### Q: 部署后第一次访问很慢？
A: Render免费层15分钟无请求会休眠，第一次访问需要唤醒（约5-10秒），之后就正常了。

### Q: Render部署失败怎么办？
A: 看一下日志里的错误信息，常见原因：
- 环境变量没填对（特别是 DB_PASSWORD）
- requirements.txt 里的包名写错了
- 把错误信息发给我，我帮你看

### Q: 怎么修改题库或添加新题？
A: 
1. 在本地修改好，更新到GitHub
2. Render 会自动检测到更新并重新部署
3. 数据库里的题目可以直接在 Supabase 的 Table Editor 里增删改

### Q: 可以改系统名字或Logo吗？
A: 可以，修改 `templates/` 里的HTML文件和 `static/css/style.css` 里的样式即可。

### Q: 手机怎么访问？
A: 手机浏览器直接输入你的Render网址就行，界面是自适应的。

### Q: 5个人同时用会不会卡？
A: 不会。答题系统的并发很低，Render免费层的512MB内存完全够用。
